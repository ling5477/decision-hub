# 目标

为 Python 模块建立可重复执行的回归测试，保护关键输入输出和边界行为。

# 适用范围

适用于：

- 工具函数测试
- 数据处理逻辑测试
- 批处理脚本核心逻辑测试
- bugfix 回归测试
- 参数校验测试
- 文件输入输出测试

不适用于：

- 前端 E2E
- 性能压测
- 与 Python 无关的系统测试

# 必须遵守

1. 优先测真实业务行为。
2. bug 修复必须补回归测试。
3. 输入输出要小而稳定。
4. 测试需可重复执行。
5. 必须覆盖正常路径和关键边界。
6. 文件/时间/随机数依赖要固定或隔离。
7. 输出必须说明覆盖点和空白点。

# 执行步骤

1. 明确测试目标
2. 划分正常路径与异常路径
3. 准备最小测试样本
4. 编写 pytest 用例
5. 执行并总结结果

# 输出格式

每次使用本 skill，输出必须按下面结构组织：

1. 测试目标
2. 覆盖范围
3. 新增/修改的测试文件
4. 关键用例说明
5. 执行结果
6. 未覆盖项

# 禁止事项

- 禁止只测 happy path
- 禁止用大样本掩盖问题
- 禁止测试依赖外部不稳定环境
- 禁止不补 bug 回归

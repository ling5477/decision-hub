# 目标

编写可靠、可读、可重复执行的 Python 脚本，用于后端辅助处理，而不是写一次性不可维护脚本。

# 适用范围

适用于：

- 数据修复脚本
- 批量导入导出
- 历史数据清洗
- 文件处理
- 离线统计
- 迁移辅助脚本
- 运维辅助脚本

不适用于：

- 长期在线服务架构设计
- 前端自动化
- 复杂平台型工程

# 必须遵守

1. 先明确输入、处理、输出。
2. 脚本必须支持最小必要参数化。
3. 必须考虑异常处理和失败重试策略。
4. 必须考虑幂等性或重复执行影响。
5. 大批量处理必须考虑分批、日志和进度输出。
6. 禁止把凭据硬编码进脚本。
7. 必须说明运行方式和前置依赖。
8. 输出结果必须可核对。
9. 对写操作必须有保护开关或明确确认策略。

# 输出格式

每次使用本 skill，输出必须按下面结构组织：

1. 脚本目标
2. 输入输出说明
3. 运行方式
4. 新增文件
5. 核心逻辑说明
6. 风险点
7. 验证方式

# 禁止事项

- 禁止把配置写死
- 禁止无日志
- 禁止批量写操作无保护
- 禁止只为跑一次就忽略可维护性
